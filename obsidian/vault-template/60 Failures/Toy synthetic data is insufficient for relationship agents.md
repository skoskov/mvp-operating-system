# Toy synthetic data is insufficient for relationship agents

## Failure

Simple clean synthetic contacts do not model a real human address book or relationship notebook.

## Why it matters

Relationship agents must handle messy, incomplete, stale, duplicate, conflicting, multilingual, and context-heavy personal data.

## New default

For relationship/contact-support MVPs, require a realistic personal-notebook simulator before feature work continues.

## Must model

- weak ties;
- stale facts;
- duplicate identities;
- ambiguous names;
- commitments;
- relationship context;
- interaction history;
- sensitive notes;
- conflicting facts;
- incomplete data.
