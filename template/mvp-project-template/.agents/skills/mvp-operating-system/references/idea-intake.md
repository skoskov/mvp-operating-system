# Idea Intake

Use Idea Intake as the first mandatory step for every new MVP or project restart.

## Purpose

Turn a raw idea into a compact, agent-readable starting point before Launch Market Research, Memory Preflight, Product/Technical Brief, OpenSpec, or implementation.

Idea Intake includes the initial MVP scope so market research can evaluate the actual first launch version, not an abstract product idea.

## Required fields

```text
Idea:
<one short paragraph>

User:
<primary user or buyer>

Problem:
<the concrete pain or job-to-be-done>

Current workaround:
<how the user solves this now>

Why current workaround is bad:
<what is slow, expensive, unreliable, risky, inconvenient, or low-quality>

Problem frequency / urgency:
<how often this happens and why it matters now>

Trigger to try MVP:
<what would make the user try this MVP>

Main use case:
<the first scenario the MVP must support>

MVP promise:
<what the user gets from the MVP>

First success moment:
<what must happen in the first minutes for the user to feel value>

MVP scope:
<what is included in the first launch version>

Must-have:
<the minimum features or artifacts required for the first user path, first value moment, first launch, and first hypothesis test>

Explicit non-goals:
<what must not be built now>

Post-launch backlog:
<useful but non-blocking items for later>

Constraints:
<time, budget, stack, market, language, data, legal, or launch constraints>

Launch target:
<where the first users will come from or how the MVP will be shown>
```

## Rules

- If the user gives only a vague idea, create a best-effort Idea Intake and ask only for missing product-critical decisions.
- Discuss MVP scope inside Idea Intake before Launch Market Research.
- Launch Market Research must use the MVP scope as its research target.
- Do not move to OpenSpec until Idea Intake exists.
- Do not expand Idea Intake into a long product spec.
- Do not ask the human to approve long documents.
- If a missing field can be safely defaulted from memory or project context, default it and record the assumption.
