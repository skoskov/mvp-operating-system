# OpenSpec Handoff

OpenSpec is the project contract for agents, not a long document for human approval.

## Required artifacts

Create or update:

- proposal / goal
- requirements / specs
- design notes if needed
- tasks / implementation slices

## Human visibility

Do not ask the human to approve full OpenSpec files.

Show only:

- short Product/Technical Brief;
- important Decision Cards;
- final summary.

## Implementation slice requirements

Each Codex slice must include:

- goal;
- scope;
- non-goals;
- relevant files;
- expected tests/evals;
- verification command;
- stop conditions;
- review requirement.
