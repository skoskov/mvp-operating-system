# Learning Update

After every major stage, update the Obsidian LLM Knowledge Base.

## Save

- important decisions;
- failed decisions;
- defaults confirmed;
- defaults changed;
- token/time estimate vs actual;
- repeated failures;
- useful tests/evals/harness patterns;
- what not to ask the human again;
- what must always be escalated.

## Output summary

```text
Learning Update

Decisions saved:
- ...

Failures saved:
- ...

Defaults updated:
- ...

Token/time actuals:
- ...

Next-project implications:
- ...
```

## Rule

Every important human decision becomes permanent memory.

Every failed technical approach becomes permanent memory.
