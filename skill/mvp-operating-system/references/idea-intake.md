# Idea Intake

Use Idea Intake as the first mandatory step for every new MVP or project restart.

## Purpose

Turn a raw idea into a compact, agent-readable starting point before Memory Preflight, Product/Technical Brief, OpenSpec, or implementation.

## Required fields

```text
Idea:
<one short paragraph>

User:
<primary user or buyer>

Problem:
<the concrete pain or job-to-be-done>

Main use case:
<the first scenario the MVP must support>

MVP promise:
<what the user gets from the MVP>

First success moment:
<what must happen in the first minutes for the user to feel value>

Non-goals:
<what must not be built now>

Constraints:
<time, budget, stack, market, language, data, legal, or launch constraints>

Launch target:
<where the first users will come from or how the MVP will be shown>
```

## Rules

- If the user gives only a vague idea, create a best-effort Idea Intake and ask only for missing product-critical decisions.
- Do not move to OpenSpec until Idea Intake exists.
- Do not expand Idea Intake into a long product spec.
- Do not ask the human to approve long documents.
- If a missing field can be safely defaulted from memory or project context, default it and record the assumption.
